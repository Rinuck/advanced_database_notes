# Session 2026-3-03

## Topics covereded
During this session we studied SQL Window Functions (also called Analytic Functions) and how they allow calculations across multiple rows while still keeping the detail of each row in the result set. Unlike traditional aggregate functions, window functions do not collapse rows, which makes them useful for analytical queries.We explored the OVER() clause, which defines the window of data that a function operates on. This clause can be empty or include additional instructions to control how the data is grouped and ordered.
Another important topic was data partitioning using PARTITION BY, which divides the dataset into logical groups (for example by department or category). Each partition is processed independently while still displaying all rows in the result. We also analyzed data ordering using ORDER BY within the OVER() clause, which determines the sequence of rows for calculations such as rankings and cumulative totals. The session also covered ranking and numbering functions, including ROW_NUMBER(), RANK(), and DENSE_RANK(). These functions are used to assign positions to rows based on ordering rules and handle ties in different ways.Finally, we studied cumulative calculations and window frames, using expressions such as ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW to define the boundaries of the window and calculate running totals or moving calculations.



## Sql overview

Window Functions: These allow for complex calculations across a set of rows while keeping individual row details visible.

The OVER() Clause: The core requirement that defines the "window" of data; it can be empty or contain specific instructions.

Partitioning and Sorting: PARTITION BY groups data into chunks, and ORDER BY defines the sequence for calculations like rankings or totals.

Windowing Frames: Using ROWS BETWEEN to set precise boundaries for calculations, such as cumulative sums.


## what i understood
Function vs. Aggregate: Unlike aggregates, window functions return a value for every single row instead of collapsing them.

Ranking Logic: ROW_NUMBER() gives a unique ID, RANK() leaves gaps during ties, and DENSE_RANK() provides continuous numbering.

Running Totals: Combining SUM() with OVER(ORDER BY...) creates a cumulative total that grows as you move down the list.

Frame Control: UNBOUNDED PRECEDING AND CURRENT ROW is the standard way to define a sliding window for running totals.
